Отчет по лабораторной работе №4
Тема: "Три толстяка"

Выполнила: Быкова Анастасия, 3 МО, РБД-1

Цель работы
Изучение механизмов многопоточного программирования в C++, синхронизации потоков с использованием мьютексов и организации взаимодействия между параллельно выполняющимися потоками на примере задачи о трех толстяках.

Общая логика программы
Участники процесса
Повар (1 поток) - готовит наггетсы и раскладывает их по 3 тарелкам

Толстяки (3 потока) - едят наггетсы из своих тарелок

Исходные данные
Начальное количество наггетсов в каждой тарелке: 3000

Лимит насыщения для толстяка: 10000 наггетсов

Продолжительность симуляции: 5 дней (5 секунд реального времени)

Ключевые параметры
GLUTTONY - коэффициент обжорства (сколько наггетсов толстяк съедает за один раз)

EFFICIENCY_FACTOR - коэффициент производительности повара (сколько наггетсов добавляется на каждую тарелку за одну итерацию)

Синхронизация потоков
Используемые механизмы
std::mutex - для защиты разделяемых данных (тарелок и счетчиков съеденного)

Атомарные переменные (std::atomic) - для безопасного обмена флагами между потоками

Строгая очередность действий - реализована через систему флагов:

cook_can_work - разрешение для повара готовить

fatmen_can_eat - разрешение для толстяков есть

fatmen_done - счетчик, сколько толстяков уже поели

Алгоритм синхронизации
Повар блокирует мьютекс, добавляет наггетсы на все тарелки, устанавливает флаги для передачи хода толстякам

Каждый толстяк в своей очереди блокирует мьютекс, ест наггетсы, увеличивает счетчик fatmen_done

Когда все три толстяка поели (fatmen_done >= 3), флаги меняются для передачи хода повару

Цикл повторяется
